# Projeto - Gestão de Configuração de Software

## 📌 Descrição

Este projeto foi desenvolvido com o objetivo de aplicar conceitos de Gestão de Configuração e Controle de Versão utilizando Git e GitHub em um ambiente colaborativo.

O cenário simula um time de desenvolvimento de software que recebe múltiplas solicitações de alteração por diferentes stakeholders, exigindo controle eficiente dos artefatos produzidos, como código-fonte, documentação e arquivos de design.

## 🎯 Objetivos

- Implementar controle de versão utilizando Git
- Utilizar repositório remoto como fonte principal dos artefatos
- Permitir colaboração entre membros do time
- Controlar histórico de alterações
- Executar merges quando necessário

## 🛠 Tecnologias Utilizadas

- Git
- GitHub
- Editor de código (VS Code ou similar)

## 📂 Estrutura do Projeto

- README.md → Documentação principal do projeto
- documentacao.txt → Documento complementar criado na prática
- /src → Pasta destinada aos arquivos de código (caso aplicável)

## 🔄 Fluxo de Trabalho

1. Clone do repositório remoto
2. Criação ou modificação de artefatos
3. Commit das alterações
4. Push para o repositório remoto
5. Pull para sincronização com outras alterações do time

## 👥 Colaboração

Todos os membros do time devem:
- Atualizar o repositório antes de iniciar alterações (git pull)
- Realizar commits descritivos
- Resolver conflitos de merge quando necessário
- Manter a documentação atualizada
